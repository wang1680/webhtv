# AI 智能去广告功能 - 功能概览

## 🎯 核心功能

用户在播放视频时发现广告 → 点击"有广告"按钮 → AI 自动分析生成去广规则 → 用户可在设置中管理规则

## 🤖 AI 工作原理

### 收集的信息
AI 分析时会收集以下信息：
1. **当前播放 URL**: 正在播放的视频地址
2. **视频标题**: 当前视频的名称
3. **站点名称**: 视频来源站点
4. **历史 URL**: 最近播放的 URL 列表（用于模式分析）

### AI 返回的规则
AI 会返回以下规则：
- **规则名称**: 建议的规则名称（如"某某视频站"）
- **域名列表**: 识别出的广告域名
- **正则表达式**: URL 匹配模式
- **排除规则**: 避免误杀的排除模式
- **分析理由**: AI 的判断依据（可选）

### 规则自动应用
AI 生成的规则会：
1. 自动保存到用户规则库
2. 立即生效（无需重启应用）
3. 标记为"AI 规则"（source="ai"）
4. 默认启用状态

## 📊 规则管理

### 规则类型

#### 1. 默认规则
- **来源**: 应用内置
- **数量**: 若干条（开发者维护）
- **操作**: 可禁用/启用，**不可删除**
- **更新**: 应用更新时自动更新

#### 2. AI 规则
- **来源**: AI 自动生成
- **标识**: source="ai"
- **操作**: 可编辑、删除、禁用
- **管理**: 用户完全控制

#### 3. 手动规则
- **来源**: 用户手动添加
- **标识**: source="manual"
- **操作**: 可编辑、删除、禁用
- **管理**: 用户完全控制

### 规则优先级

```
用户启用的规则（AI + 手动）> 默认规则
```

### 规则匹配逻辑

```python
def should_block_url(url):
    for rule in all_enabled_rules:
        # 1. 检查域名匹配
        if matches_hosts(url, rule.hosts):
            # 2. 检查排除规则
            if not matches_exclude(url, rule.exclude):
                return True  # 拦截
        
        # 3. 检查正则匹配
        if matches_regex(url, rule.regex):
            # 4. 检查排除规则
            if not matches_exclude(url, rule.exclude):
                return True  # 拦截
    
    return False  # 放行
```

**任何规则的排除字段都可以覆盖其他规则的匹配结果。**

```json

  "version": 1,

      "name": "AI 规则：某某视频站",
      "hosts": [
        "adv.example.com",
        "*.ad.provider.net"
      ],
      "regex": [
        ".*\\.mp4\\?ad=.*",
        ".*/preroll/.*"
      ],
      "exclude": [
        ".*trailer.*"
      ],
      "enabled": true,
      "createdAt": 1720569600000,
      "source": "ai"
    }
  ]
}
```

## ❓ 常见问题

### Q: "有广告"按钮不显示？
**A**: 检查以下条件：
1. 设置中"AI 功能"是否已开启
2. 设置中"去广功能"是否已开启
3. 两者必须**同时开启**

### Q: AI 分析失败怎么办？
**A**: 可能原因：
- 网络连接问题
- AI 服务暂时不可用
- 视频信息不足以分析

**解决方案**: 稍后重试，或手动添加规则

### Q: 规则会误拦截正常视频吗？
**A**: 可能性：
- AI 可能生成过于宽泛的规则
- 某些站点的正片 URL 模式与广告相似

**解决方案**:
- 在规则编辑中添加"排除规则"
- 禁用或删除误拦的规则

### Q: 默认规则可以删除吗？
**A**: 不可以。默认规则由开发者维护，仅支持禁用/启用。

### Q: 规则数量有限制吗？
**A**: 当前版本无硬性限制，但建议：
- 用户规则 < 50 条（性能考虑）
- 定期清理不再使用的规则

### Q: 规则会同步到其他设备吗？
**A**: 当前版本不支持跨设备同步。未来可通过备份/恢复功能实现。
